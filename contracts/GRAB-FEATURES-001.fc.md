# Feature Contract: Contract-Backed Feature Index
**ID:** GRAB-FEATURES-001 | **Status:** completed
CONTRACT_TYPE: FEATURE_CONTRACT
ARCH_VERSION: v1
RULESET_VERSION: v1
ENV_VERSION: v1

## Objective
Implement repo-native feature tracking from `contracts/*.fc.md`, add feature index commands, and deprecate standalone ticket markdown so contracts become the canonical engineering artifact.

## Scope
- Add `grabby features:list`
- Add `grabby features:status <ID>`
- Add `grabby features:refresh` to regenerate `.grabby/features.index.json`
- Parse feature metadata from contracts and derived plan/audit artifacts
- Warn on deprecated standalone ticket markdown during validation
- Update docs to state that feature contracts are canonical

## Non-Goals
- Auto-deleting legacy ticket files
- Expanding scope into Jira API behavior changes
- Removing existing contract/plan/audit artifact types

## Directories
**Allowed:** `bin/`, `lib/`, `docs/`, `tests/`, `contracts/`, `.grabby/`
**Restricted:** `node_modules/`, `dist/`, `.git/`
