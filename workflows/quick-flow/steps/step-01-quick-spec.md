# Step 1: Quick Spec

## Goal
Rapidly capture the essentials for a small change.

## Quick Questions

Only 4 questions needed:

### 1. What's the change?
```
Describe the change in one sentence:
> ___________________________________
```

### 2. Which files?
```
Which files will be touched? (max 3)
> ___________________________________

Example: src/hooks/useAuth.ts, src/types.ts
```

### 3. What's the test?
```
How will we know it works?
> ___________________________________

Example: Login redirects to dashboard
```

### 4. Any risks?
```
Any gotchas or risks? (or "none")
> ___________________________________
```

## Quick Validation

Before proceeding, verify:
- [ ] 3 or fewer files
- [ ] Single, clear objective
- [ ] No architectural changes
- [ ] No new dependencies

If any fail:
```
⚠ This change may be too large for quick flow.
Consider: grabby agent architect CC

[C] Continue anyway
[E] Escalate to full contract
```

## Navigation
- [C] Continue to confirmation
- [Q] Quit
