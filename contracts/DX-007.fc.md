# FC: Consolidate color codes and output formatting
**ID:** DX-007 | **Status:** draft
**Type:** refactor

## Ticket
- Who: Grabby maintainers
- What: Consolidate ANSI color codes into single module
- Why: Color codes defined in 3+ places (commands.cjs, tui.cjs, bin/index.cjs)

## Objective
Create `lib/colors.cjs` with centralized ANSI color utilities and update all usages across the codebase.

## Scope
- Extract color codes to `lib/colors.cjs`
- Create semantic color functions (success, error, warn, info, dim)
- Create formatting helpers (bold, underline, code)
- Replace hardcoded ANSI sequences across codebase
- Add NO_COLOR environment variable support

## Non-Goals
- No new visual styles
- No third-party color libraries

## Directories
**Allowed:** `lib/`, `bin/`, `tests/`
**Restricted:** `node_modules/`

## Files
| Action | Path | Reason |
|--------|------|--------|
| create | `lib/colors.cjs` | Centralized color utilities |
| create | `tests/colors.test.js` | Unit tests |
| modify | `lib/commands.cjs` | Use colors module |
| modify | `lib/tui.cjs` | Use colors module |
| modify | `bin/index.cjs` | Use colors module |
| modify | `lib/interactive-shell.cjs` | Use colors module |

## Dependencies
- No dependencies (pure ANSI escape codes)

## Done When
- [ ] `lib/colors.cjs` created with success/error/warn/info/dim/bold
- [ ] NO_COLOR env support implemented
- [ ] No hardcoded `\x1b[` or `\033[` in other files
- [ ] All colored output uses colors module
- [ ] Tests verify color output and NO_COLOR handling

## Testing
- Unit: `tests/colors.test.js`
- Verify: `NO_COLOR=1 grabby list` shows no color codes
