# Step 1: Structure Validation

## Goal
Verify the contract has all required sections with proper formatting.

## Required Sections
Check for presence and content:

1. **Objective** (required)
   - Must exist
   - Should be 1-2 sentences
   - Should be concrete and measurable

2. **Scope** (required)
   - Must have at least one item
   - Items should be actionable

3. **Directories** (required)
   - Must specify Allowed directories
   - Must specify Restricted directories

4. **Files** (required)
   - Must be a table with Action | Path | Reason
   - Each row must have all columns

5. **Done When** (required)
   - Must have verifiable criteria
   - Should be checkboxes

## Recommended Sections
Check and warn if missing:

6. **Non-Goals** - Helps prevent scope creep
7. **Testing** - Ensures quality
8. **Context Refs** - Helps AI implementation

## Output
Report structure validation results:
- Pass/Fail for each required section
- Warning for missing recommended sections
- Specific issues found
