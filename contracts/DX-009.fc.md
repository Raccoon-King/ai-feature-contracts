# FC: Add config schema validation
**ID:** DX-009 | **Status:** draft
**Type:** enhancement

## Ticket
- Who: Grabby users with custom configs
- What: Validate config files against schema before use
- Why: Silent failures on bad config cause confusing downstream errors

## Objective
Add JSON schema validation to `lib/config.cjs` so invalid configurations fail fast with clear error messages.

## Scope
- Define JSON schema for grabby.config.json
- Validate config on load
- Report all validation errors at once (not just first)
- Add schema documentation to docs/

## Non-Goals
- No migration tooling
- No config editor UI
- No breaking changes to valid configs

## Directories
**Allowed:** `lib/`, `docs/`, `tests/`
**Restricted:** `node_modules/`

## Files
| Action | Path | Reason |
|--------|------|--------|
| create | `lib/config-schema.cjs` | Config JSON schema |
| create | `docs/CONFIG_SCHEMA.md` | Schema documentation |
| modify | `lib/config.cjs` | Add validation on load |
| modify | `tests/config.test.js` | Test validation errors |

## Dependencies
- Allowed: ajv (JSON schema validator) or hand-rolled validation
- Prefer hand-rolled to avoid new dependency

## Done When
- [ ] Schema defined for all config options
- [ ] Invalid configs fail with clear error listing all issues
- [ ] Valid configs load without changes
- [ ] docs/CONFIG_SCHEMA.md documents all options
- [ ] Tests cover valid, invalid, and edge case configs

## Testing
- Unit: tests/config.test.js with invalid config fixtures
- Manual: Introduce typo in config, verify clear error
