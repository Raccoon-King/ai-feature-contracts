# FC: Audit and Update Grabby Rulesets Repository
**ID:** FC-1774138747364 | **Status:** approved
**Data Change:** no
**API Change:** no

## Ticket
- Who: Grabby maintainers
- What: Audit and update the central Grabby Rulesets Repository
- Why: Ensure rulesets accurately reflect real-world project standards and provide actionable governance

## Objective
Perform a comprehensive audit of the Grabby Rulesets Repository (https://github.com/Raccoon-King/grabby-contracts.git), review all ruleset definitions, enhance rules for Node.js/Express/Jest stack, fix any schema issues, and ensure the manifest is properly structured.

## Background
The central rulesets repository provides governance rules that Grabby syncs to local projects. Current state:
- 19 rulesets across 6 categories
- Manifest version 1.0.0
- Rules need validation against actual project patterns

## Scope

### Phase 1: Repository Audit
- Clone and analyze current ruleset structure
- Document all existing rules and their coverage
- Identify gaps in Node.js/Express/Jest rules
- Validate manifest.yaml schema

### Phase 2: Language Rulesets Update
- **languages/javascript** - Enhance for modern CommonJS patterns
  - Module import/export conventions
  - Error handling patterns
  - Async/await best practices
  - No TypeScript dependency assumptions

### Phase 3: Framework Rulesets Update
- **frameworks/express** - Add comprehensive Express rules
  - Middleware ordering
  - Error handler placement
  - Route organization
  - Security middleware requirements (helmet, cors, rate-limit)
  - Request validation patterns

### Phase 4: Testing Rulesets Update
- **testing/unit** - Jest-specific rules
  - Test file naming conventions
  - Mock patterns
  - Assertion best practices
  - Coverage thresholds
- **testing/coverage** - Coverage enforcement
  - Minimum thresholds (80% line/branch/function)
  - Exclusion patterns
  - CI integration requirements

### Phase 5: Domain Rulesets Update
- **domains/api-compat** - REST API rules
  - Versioning requirements
  - OpenAPI spec requirements
  - Breaking change detection
  - Response format standards

### Phase 6: Policy Rulesets Update
- **policies/security** - Security checklist
  - Input validation
  - Output encoding
  - Authentication requirements
  - Secret management
- **policies/git-workflow** - Git rules
  - Branch naming
  - Commit message format
  - PR requirements

### Phase 7: Manifest & Documentation
- Validate and fix manifest.yaml structure
- Update ruleset descriptions
- Add usage examples
- Create ruleset authoring guide

## Non-Goals
- Creating new ruleset categories
- Changing the ruleset sync protocol
- Modifying Grabby core to handle new ruleset formats
- TypeScript-specific rules (focus on JavaScript/CommonJS)
