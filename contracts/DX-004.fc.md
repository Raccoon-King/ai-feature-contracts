# FC: Split commands.cjs into modules
**ID:** DX-004 | **Status:** draft
**Type:** refactor

## Ticket
- Who: Grabby maintainers
- What: Decompose 2,936-line commands.cjs monolith into focused modules
- Why: Single file with 44+ functions is hard to navigate and maintain

## Objective
Split `lib/commands.cjs` into domain-specific modules under `lib/commands/` while maintaining backward compatibility through re-exports.

## Scope
- Create `lib/commands/` directory with domain-specific modules
- Extract commands into: task.cjs, validate.cjs, plan.cjs, execute.cjs, audit.cjs, init.cjs, git.cjs
- Keep `lib/commands.cjs` as facade re-exporting all modules

## Non-Goals
- No behavior changes
- No new commands
- No API changes to exported functions

## Directories
**Allowed:** `lib/`, `tests/`
**Restricted:** `node_modules/`, `.git/`
