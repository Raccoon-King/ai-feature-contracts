## Ticket
- Who: Grabby maintainers and operators
- What: Refine agent routing to select the best specialist by stage and substep
- Why: Current keyword-heavy routing can misroute complex work; we need deterministic, state-aware handoffs

## Objective
Implement state-aware, substep-aware, confidence-aware routing so Grabby selects the best agent at each step/substep while preserving governance gates

## Scope
- Replace keyword-first routing with state-first routing
- Enforce artifact governance gates in routing
- Add substep-aware routing model
- Add confidence scoring and fallback routing
- Add routing memory and outcome feedback
- Add re-evaluation checkpoints after major transitions
- Expand tests for known misrouting scenarios and decision-trace observability

## Non-Goals
- Add new end-user product features
- Introduce multi-writer concurrent execution
- Modify unrelated workflows
