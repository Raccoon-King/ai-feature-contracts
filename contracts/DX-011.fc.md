# FC: Configurable validation rules
**ID:** DX-011 | **Status:** draft
**Type:** feat

## Ticket
- Who: Grabby users with custom governance needs
- What: Make validation rules configurable (error vs warning vs disabled)
- Why: Rules are hardcoded in core.cjs - no way to customize per-project

## Objective
Extract validation rules into a configurable schema so projects can promote warnings to errors, demote errors to warnings, or disable rules entirely.

## Scope
- Extract rules from core.cjs into rule definitions
- Create rule schema with id, level (error/warn/off), message
- Add `validation.rules` config section to grabby.config.json
- Support rule overrides per-project
- Keep sensible defaults (current behavior)

## Non-Goals
- No custom rule creation (just level configuration)
- No per-contract rule overrides
- No UI for rule management

## Directories
**Allowed:** `lib/`, `tests/`, `docs/`
**Restricted:** `node_modules/`

## Files
| Action | Path | Reason |
|--------|------|--------|
| create | `lib/validation-rules.cjs` | Rule definitions and engine |
| modify | `lib/core.cjs` | Use validation-rules engine |
| modify | `lib/config.cjs` | Add validation.rules schema |
| create | `tests/validation-rules.test.js` | Unit tests |
| modify | `docs/CONFIG_SCHEMA.md` | Document rule config |

## Rule Definition Example
```javascript
const RULES = {
  'missing-section': {
    id: 'missing-section',
    default: 'error',
    check: (content, section) => content.includes(`## ${section}`),
    message: (section) => `Missing section: ${section}`,
  },
  'scope-too-large': {
    id: 'scope-too-large',
    default: 'error',
    threshold: 7,
    check: (content) => countScopeItems(content) <= 7,
    message: (count) => `Scope too large (${count} items) - max 7 recommended`,
  },
  'vague-objective': {
    id: 'vague-objective',
    default: 'warn',
    check: (content) => !VAGUE_TERMS.some(t => getObjective(content).includes(t)),
    message: (term) => `Vague term in Objective: "${term}"`,
  },
};
```

## Config Example
```json
{
  "validation": {
    "rules": {
      "scope-too-large": "warn",
      "missing-test-file": "error",
      "vague-objective": "off"
    }
  }
}
```
