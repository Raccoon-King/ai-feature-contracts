# FC: Developer involvement modes
**ID:** DX-010 | **Status:** draft
**Type:** feat

## Ticket
- Who: Developers using grabby with AI assistants
- What: Add configurable developer involvement modes for contract execution
- Why: Different devs want different levels of control - some want full automation, others want to write code themselves with AI guidance

## Objective
Add three developer involvement modes that control how much the AI does vs explains during contract execution.

## Modes

### 1. YOLO Mode (`--mode yolo`)
- Dev approves the contract/plan only
- AI implements everything autonomously
- Dev reviews final PR
- Best for: Trusted patterns, boilerplate, simple features
- Output: Code changes + summary

### 2. Guided Mode (`--mode guided`) [Default]
- AI proposes changes file-by-file
- Dev reviews each change before applying
- AI explains "why" for each change
- Dev can modify, skip, or request alternatives
- Best for: Learning codebase, moderate complexity
- Output: Interactive prompts + code changes

### 3. Mentor Mode (`--mode mentor`)
- AI explains the approach and logic only
- AI shows pseudocode/structure, not implementation
- Dev writes all code themselves
- AI answers questions and reviews dev's code
- Best for: Skill building, complex/critical code
- Output: Explanation + guidance prompts

## Scope
- Add `--mode` flag to `grabby execute`
- Add `devMode` config option in grabby.config.json
- Implement YOLO mode (current behavior, minimal changes)
- Implement Guided mode with file-by-file prompts
- Implement Mentor mode with explanation-only output
- Add mode indicator in execution output

## Non-Goals
- No changes to contract/plan/audit phases
- No per-file mode switching (whole execution uses one mode)
- No IDE integration (CLI only)

**Allowed:** `lib/`, `tests/`, `docs/`
**Restricted:** `node_modules/`

## CLI Examples
```bash
grabby execute FOO-001.fc.md --mode yolo    # Full automation
grabby execute FOO-001.fc.md --mode guided  # Interactive review
grabby execute FOO-001.fc.md --mode mentor  # Explanation only
```
