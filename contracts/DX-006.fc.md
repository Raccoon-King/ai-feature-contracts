# FC: Improve error messages with context
**ID:** DX-006 | **Status:** draft
**Type:** enhancement

## Ticket
- Who: Grabby users
- What: Add actionable context to error messages
- Why: "Contract not found" doesn't help users understand what went wrong or how to fix it

## Objective
Enhance error messages across grabby to include file paths searched, suggested next commands, and recovery hints.

## Scope
- Add searched paths to "not found" errors
- Add suggested commands to common errors
- Add recovery hints for fixable errors
- Standardize error message format across modules

## Non-Goals
- No new error types
- No logging infrastructure changes
- No exit code changes

## Directories
**Allowed:** `lib/`
**Restricted:** `tests/`, `node_modules/`

## Files
| Action | Path | Reason |
|--------|------|--------|
| modify | `lib/features.cjs` | Enhance feature not found errors |
| modify | `lib/commands.cjs` | Enhance command errors |
| modify | `lib/core.cjs` | Enhance validation errors |
| modify | `lib/config.cjs` | Enhance config errors |

## Error Message Format
```
Error: Contract 'FOO-001' not found
  Searched: contracts/, contracts/active/, .grabby/contracts/
  Suggestion: Run 'grabby list' to see available contracts
```

- No dependencies
