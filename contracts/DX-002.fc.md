# FC: Async-ify API server I/O
**ID:** DX-002 | **Status:** draft
**Type:** refactor

## Ticket
- Who: API consumers and Grabby maintainers
- What: Replace synchronous file I/O with async operations in api-server
- Why: Synchronous fs.readdirSync/readFileSync in request handlers blocks all requests

## Objective
Convert `lib/api-server.cjs` to use async filesystem operations so large contract directories don't block the request thread.

## Scope
- Replace `fs.readdirSync` with `fs.promises.readdir`
- Replace `fs.readFileSync` with `fs.promises.readFile`
- Replace `fs.writeFileSync` with `fs.promises.writeFile`
- Add proper async/await error handling
- Add timeout handling for file operations

## Non-Goals
- No changes to other lib files (they remain sync for CLI use)
- No new API endpoints
- No caching layer (separate contract)

## Directories
**Allowed:** `lib/`, `tests/`
**Restricted:** `node_modules/`, `.git/`

## Files
| Action | Path | Reason |
|--------|------|--------|
| modify | `lib/api-server.cjs` | Convert to async I/O |
| modify | `tests/api-server.test.js` | Update tests for async |

## Dependencies
- Allowed: existing packages only
- Use `fs.promises` (built-in)

## Done When
- [ ] All request handlers use async file operations
- [ ] No `fs.*Sync` calls remain in api-server.cjs
- [ ] File operation timeouts implemented (5s default)
- [ ] Error responses include operation context
- [ ] Tests pass with async mocking
