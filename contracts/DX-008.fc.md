# FC: Fix N+1 file reads in features.cjs
**ID:** DX-008 | **Status:** draft
**Type:** performance

## Ticket
- Who: Users with large contract directories
- What: Eliminate redundant file reads in feature discovery
- Why: Contract parsing reads files multiple times, slowing down listing

## Objective
Optimize `lib/features.cjs` to read each contract file once and cache parsed results during discovery operations.

## Scope
- Cache file contents during directory traversal
- Parse contract ID and metadata in single pass
- Avoid re-reading files for different fields
- Add early termination for archive searches

## Non-Goals
- No persistent caching (memory only during operation)
- No async changes (separate DX-002)
- No database/index file

## Directories
**Allowed:** `lib/`, `tests/`
**Restricted:** `node_modules/`

## Files
| Action | Path | Reason |
|--------|------|--------|
| modify | `lib/features.cjs` | Optimize file reads |
| modify | `tests/features.test.js` | Add performance assertions |

## Dependencies
- No dependencies

- [ ] `discoverFeatures()` reads each file exactly once
- [ ] Archive traversal stops early when found
- [ ] Performance test: 100 contracts listed in < 500ms
